Build CI/CD Pipeline using Git, GitHub and Vercel

Git

+ Setting Up Local Project

  1. mkdir <project-name> (Create a project folder)
  2. touch index.html (Create a html file index.html)
  3. touch styles.html (Create a css file styles.css)
  4. make html boilerplate in index.html
  5. git init (make current project folder a local git repository)

+ Set Up Origin, Commit and Push

  1. git remote add origin <url-provided> (Connects local repo with GitHub repo)
  2. git add . (Adds all the changes to staging)
  3. git commit -m "Initial Commit" (Commits all the changes added in staging area)
  4. git push -u origin main (push the commits to the connected GitHub repo)

+ Create Feature Branch, Commit and Push

  1. git checkout -b <feature-branch-name> (Creates New branch)
  2. git add . (Adds all the changes to staging)
  3. git commit -m "Add new feature" (Commits all the changes added in staging area)
  4. git push origin <feature-branch-name> (push the commits to the connected GitHub repo new branch)
GitHub

+ Setting Up New Github Repo

  1. Navigate on Github > `repository`
  2. Click on `New` to create new repository
  3. Name the repository and Click `create repository`
  4. copy the `git remote add origin <url-provided>`

+ Create Pull Request and Merge Pull Request

  1. Navigate on Github Project Repo, click on Pull Request
  2. Click on `Compare & Pull Request` for the branch you want to merge
  3. Type title and description
  4. Click on `Create Pull Request`
  5. Click on `Merge`
  6. Click on `Confirm Merge` to confirm merge pull request then the branch will be merged to the main branch
Vercel

+ Deploy on Vercel

  1. Go to Vercel `Dashboard` inside `overview` section, Click on `Add New` and Select `Project`
  2. From `Import Git Repository` Section, `Import` the current project from the list
  3. Setup new project information, then Click `Deploy`
  4. Click on `Visit` to view deployed project

+ Vercel Ceontinious Depoyment

  1. To update your Production Deployment, push to the "main" branch.
  2. Everytime the pull request is merged or main branch is updated automatic deployment is triggered, which is continious Deployment
